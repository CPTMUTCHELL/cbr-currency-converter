Couldn't auth via SA at the first time, had to use personal account
1. gcloud auth login
2. Switch to created from web-console admin-sa with Compute Admin Create Service Accounts Service Account Key Admin roles

2.1 create key 
gcloud iam service-accounts keys create ~/admin-sa.json --iam-account=admin-sa@aerobic-datum-327116.iam.gserviceaccount.com
2.2 gcloud auth activate-service-account admin-sa@aerobic-datum-327116.iam.gserviceaccount.com --key-file /home/cptmutchell/admin-sa.json --project=aerobic-datum-327116

now the active account is admin-sa
2.3 revoke private account for security
gcloud auth revoke cptmutchell@gmail.com

3. Create jenkins-sa
3.1 gcloud iam service-accounts create jenkins-sa --display-name=jenkins-sa
3.2 gcloud iam service-accounts keys create ~/jenkins-sa.json --iam-account=jenkins-sa@aerobic-datum-327116.iam.gserviceaccount.com

3.3 Had to give Owner role to use iam-policy-binding
gcloud projects add-iam-policy-binding aerobic-datum-327116 --member="serviceAccount:jenkins-sa@aerobic-datum-327116.iam.gserviceaccount.com" --role="roles/container.developer"
3.4 gcloud auth activate-service-account jenkins-sa@aerobic-datum-327116.iam.gserviceaccount.com --key-file /home/cptmutchell/jenkins-sa.json --project=aerobic-datum-327116


4. update gcloud 
4.1 sudo apt-get install google-cloud-sdk-kubectl
4.2 sudo apt-get install kubectl
5. Acquire cluster creds. Now kubectl is connected to our cluster
gcloud container clusters get-credentials cluster-1 --zone europe-central2-a --project aerobic-datum-327116
